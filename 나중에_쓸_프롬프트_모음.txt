[나중에 Codex에게 요청할 수 있는 프롬프트 모음]

1. 줄 제거 데미지 팝업 추가
줄 제거로 사라진 숫자들의 합만큼 데미지가 들어갈 때, 적 위에 데미지 숫자 팝업이 뜨게 해줘. 너무 크게 바꾸지 말고 지금 전투 UI 스타일에 맞춰서 구현해줘. Inspector에서 팝업 크기, 색, 지속시간도 조절 가능하게 해줘.

2. 줄 제거 히트 이펙트 추가
줄 제거 데미지가 들어갈 때 적 캐릭터에 히트 이펙트가 나오게 해줘. 기존 피격 연출과 겹쳐도 어색하지 않게 해주고, 이펙트 오브젝트는 Inspector에서 연결 가능하게 해줘.

3. 줄 제거 사운드 추가
줄 제거가 발생하면 전용 사운드가 재생되게 해줘. 숫자 줄 제거일 때만 재생하고, AudioSource나 AudioClip은 Inspector에서 연결 가능하게 해줘.

4. 시작 아이템 선택 연출 강화
시작 아이템 버튼을 선택했을 때 아우라가 켜지는 것 말고도 버튼 확대, 색 변화, 파티클 같은 연출을 추가해줘. 너무 과하지 않게 하고 Inspector에서 강도나 시간 조절 가능하게 해줘.

5. 선택 강제용 배경 흐리기
시작 아이템 패널이 떠 있을 때 뒤 배경이 클릭 안 되는 것뿐 아니라 더 어둡거나 흐릿하게 보여서 반드시 선택해야 한다는 느낌이 들게 해줘.

6. 유니크 아이템 툴팁 개선
유니크 아이템 1~9 설명을 플레이어가 바로 이해할 수 있게 더 짧고 직관적인 문장으로 바꿔줘. 필요하면 툴팁용 짧은 설명과 상세 설명을 분리해줘.

7. Inspector로 조절 가능한 연출 늘리기
지금 코드에서 하드코딩된 연출 시간, 색상, 흔들림 강도, 딜레이 같은 값들 중 자주 만질 것들을 Inspector에서 조절 가능하게 정리해줘.

8. 현재 전투 연출 분석 요청
현재 전투 연출에서 Inspector로 빼는 게 좋은 값들, 재사용 가능한 연출들, 나중에 손보면 좋은 부분들을 분석해서 정리해줘. 코드는 바로 바꾸지 말고 먼저 분석만 해줘.

9. 데미지 연출 세트 한 번에 추가
줄 제거 데미지에 대해 데미지 숫자 팝업, 히트 이펙트, 사운드까지 한 번에 붙여줘. 기존 코드 스타일 유지하고 Inspector 연결 포인트도 같이 만들어줘.

10. 내가 원하는 방식으로 수정 요청하기
다음 작업을 해줘.
- 무엇을 바꾸고 싶은지: [여기에 작성]
- 어떤 상황에서 발동하는지: [여기에 작성]
- 어떻게 보이길 원하는지: [여기에 작성]
- Inspector에서 조절 가능해야 하는 값: [여기에 작성]
- 건드리면 안 되는 것: [여기에 작성]

예시:
다음 작업을 해줘.
- 무엇을 바꾸고 싶은지: 줄 제거 데미지 연출
- 어떤 상황에서 발동하는지: 숫자 타일이 한 줄 가득 차서 자동 제거될 때
- 어떻게 보이길 원하는지: 적 위에 데미지 숫자가 뜨고 짧게 흔들리면 좋겠음
- Inspector에서 조절 가능해야 하는 값: 팝업 시간, 폰트 크기, 색상, 흔들림 세기
- 건드리면 안 되는 것: 기존 유니크 로직, 상점 로직

11. 디버깅 요청 프롬프트
지금 [문제 상황]이 발생하고 있어. 관련 코드 찾아서 원인 분석해주고, 바로 고칠 수 있으면 수정까지 해줘. 수정 후 어떤 파일을 어떻게 바꿨는지도 같이 알려줘.

12. 분석만 먼저 요청하는 프롬프트
지금 [기능 이름] 구조를 먼저 분석해줘. 관련된 파일, 실제 동작 흐름, Inspector 연결 포인트, 수정 시 영향 범위를 먼저 정리해줘. 아직 코드는 수정하지 마.

13. 작은 수정 요청 프롬프트
지금 기능은 유지하고, [특정 부분]만 최소 수정으로 바꿔줘. 다른 시스템 영향 없게 작게 수정해줘.

14. 연출 추가 요청 프롬프트
[특정 이벤트]가 발생할 때 [원하는 연출]이 나오게 해줘. 가능하면 기존 코드 구조에 맞춰서 넣고, 세기/시간/색 같은 값은 Inspector에서 조절 가능하게 해줘.

15. UI 차단 요청 프롬프트
[특정 패널]이 떠 있는 동안 뒤쪽 UI가 눌리지 않게 해줘. 필요하면 클릭 차단, 비활성화, 어두운 오버레이 처리까지 같이 해줘.
