작업 방향
- 이번 구현은 문서 전체를 이상적으로 다 맞추는 대신, 사용자 합의에 따라 "Normal 난이도만 구현한다고 생각"하는 전제로 진행했다.
- 즉 Easy / Hard 분기와 TitleScene 기반 난이도 선택 진입은 이번 범위에서 제외하고, Normal 난이도에서 UniqueItem 시스템이 항상 활성화된 상태를 먼저 구현하는 방향으로 진행했다.

모호성 레벨이 높아 질문했던 내용과 사용자 선택

1. UniqueItem 구현 범위
- 질문 요지:
  - 문서 기준으로 필요한 기반 시스템까지 같이 구현할지
  - 아니면 현재 프로토타입 구조를 유지하면서 가능한 것만 먼저 구현할지
- 사용자 선택:
  - 1번. 문서 기준으로 필요한 기반까지 같이 구현
- 실제 진행 시 적용:
  - 다만 이후 대화에서 난이도는 Normal만 구현한다고 합의되었으므로, 최종적으로는 "문서 구조를 참고하되 Normal 전용 프로토타입 구현"으로 수렴했다.

2. 난이도 / 진입 방식
- 질문 요지:
  - TitleScene 쪽에도 난이도 선택 시작 흐름을 새로 만들지
  - 아니면 현재 BattleScene 시작 시 오버레이로 난이도/시작 Unique 선택을 붙일지
- 사용자 반응:
  - 난이도가 왜 필요한지 먼저 질문
  - 이후 Easy 난이도에서는 UniqueItem 시스템이 없다는 설명을 들은 뒤, "그러면 normal 난이도만 구현 한다고 생각하자"라고 확정
- 최종 선택:
  - Normal 난이도만 구현
  - 별도 TitleScene 난이도 선택 구현은 제외
  - BattleScene 중심으로 Unique 시스템을 붙이는 방향

3. Unique 3(성부/성자/성령)의 Attack Mode 추가 데미지 처리
- 질문 요지:
  - 결과가 0 이하인 Attack Mode에서도 추가 데미지 30을 줄지
  - 아니면 실드만 줄지
- 사용자 선택:
  - 우선 추가데미지 주는걸로 적용
- 반영 규칙:
  - Attack Mode에서 Unique 3 조건이 만족되면 base result와 별개로 보너스 데미지를 적용
  - 따라서 결과가 0이어도 보너스 데미지 적용
  - 결과가 음수여도 적 회복 처리와 별개로 보너스 데미지 적용

4. Attack / Defense Mode 유지 방식
- 질문 요지:
  - 버튼을 한 번 누르면 계속 유지되는 토글형인지
  - 아니면 유효 수식 1회마다 다시 선택하는 방식인지
- 사용자 선택:
  - 1번 토글 유지 방식
- 반영 규칙:
  - Attack / Defense 버튼으로 현재 모드를 바꾸고
  - 사용자가 다시 누르기 전까지 유지

5. 실드 누적 규칙 1차 질문
- 질문 요지:
  - 적 공격 전에 여러 번 실드를 얻으면 계속 누적할지
  - 아니면 최근/더 적절한 값으로 덮어쓸지
- 사용자 선택:
  - 비교해서 더 높은 수식이 적용 되게 하자

6. 실드 누적 규칙 2차 정정
- 사용자 정정:
  - "아니 unique 실드는 그냥 주고 기본 값 실드만 비교해서 적용되게"
- 최종 확정 규칙:
  - 기본 Defense 실드만 기존 실드와 비교해서 더 높은 값을 유지
  - Unique 실드 보너스는 비교에 넣지 않고 별도로 그냥 추가

최종 실드 규칙 정리
- Defense Mode 기본 실드:
  - 기존 실드와 새 기본 실드를 비교해서 더 높은 값 유지
- Unique 3 / Unique 5 등으로 얻는 실드:
  - 기본 실드 비교와 별개로 추가 지급
- Attack Mode에서 발생하는 Unique 실드:
  - 현재 실드에 바로 추가
- 적 공격 후:
  - 남은 실드는 0으로 초기화

우선적으로 진행했던 구현 영역

1. 시작 Unique 선택
- 게임 시작 시 UniqueItem 3개 후보를 보여주고 1개를 선택하는 흐름을 먼저 붙였다.
- Normal 난이도 고정 전제를 기준으로 항상 활성화되도록 진행했다.

2. Unique 상점 슬롯
- Stage 3 / 6 / 9 상점에서 Unique Item Slot이 등장하도록 우선 구현했다.
- 기존 상점 흐름 안에 유니크 슬롯을 끼워 넣는 방식으로 진행했다.

3. Attack / Defense 모드 + 실드 시스템
- UniqueItem 문서가 요구하는 효과를 넣기 위해, 먼저 전투 쪽 기반 시스템으로 Attack / Defense 토글과 Shield 개념을 추가했다.

4. Unique 1~9 데이터/효과 연결
- 문서 기준으로 Unique 1~9를 식별 가능한 데이터로 추가하고,
- 현재 전투/상점 구조에 붙일 수 있는 효과부터 우선 연결했다.

5. 보드 생성 / 전투 결과 / 스테이지 보상 쪽 규칙 연결
- Unique 4: 보드 생성 비율
- Unique 6 / 8: 스테이지 클리어 골드
- Unique 9: 특정 턴 보드 변환
- Unique 2 / 3 / 5 / 7: 전투 결과 후처리

현재 전제
- 이번 구현은 Normal 난이도 전용 가정이다.
- Easy / Hard 분기, TitleScene 진입, 전체 난이도 UX는 아직 이번 범위에 포함하지 않았다.
- 모호성이 큰 부분은 질문 후 결정했고, 그 외는 문서에 맞춰 우선 구현 가능한 부분부터 진행했다.
