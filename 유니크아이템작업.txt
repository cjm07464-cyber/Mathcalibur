- 유니크 가격은 데이터에서 가져오도록 연결

11. 보드 생성 관련 작업

11-1. Unique 4 연결
- 타일 생성 시 숫자/연산자 비율 계산에 Unique 4 반영
- A셀과 B셀에 대해 각각 다른 비율 적용
- 비율 계산은
  - numberRatio / (numberRatio + operatorRatio)
  방식으로 수정
- 즉 90/10, 10/90 같은 문서 값이 보다 정확하게 반영되도록 정리

12. 수식 평가 / 선택 흐름 재구성

  - 수식 결과 계산

13. Unique별 구현 내용

13-1. Unique 1
- 스테이지 동안 사용한 숫자 1의 개수 추적
- 일정 수 이상 사용하면 다음 유효 수식에서 계산용 1을 11로 변환
- Unique 9가 먼저 보드를 바꾸면, 1이 9로 바뀐 경우 11로 되지 않도록 우선순위 정리

13-2. Unique 2
- Attack Mode에서만 동작
- baseResult > 0 일 때만 동작
- 수식 내 숫자 2의 개수에 따라 확률 증가
- 최대 확률 제한 있음
- 발동 시 base damage 비율 기반 추가 데미지

13-3. Unique 3
- 최종 숫자들에 3, 6, 9가 모두 포함되면 발동
- Attack Mode:
  - 추가 데미지 부여
  - 추가 실드 부여
- Defense Mode:
  - 방어 실드 보너스 부여
- 사용자 결정에 따라 결과가 0 이하인 Attack Mode에서도 추가 데미지 적용

13-4. Unique 4
- 보드 숫자/연산자 생성 비율 오버라이드
- A/B 셀 비율을 별도 설정값으로 사용

13-5. Unique 5
- 수식 안의 숫자 5 개수에 비례해 실드 추가

13-6. Unique 6
- 스테이지 클리어 시 고정 골드 보너스 추가

13-7. Unique 7
- Attack Mode에서만 동작
- 수식 길이가 정확히 7일 때 추가 데미지 비율 보너스

13-8. Unique 8
- 스테이지 클리어 골드에 퍼센트 배율 적용

13-9. Unique 9
- 특정 유효 턴에서 보드 전체를 검사
- 지정 값 이하 숫자 타일을 target value 로 변환
- 변환 후 수식 컨텍스트를 다시 구성하므로 Unique 3 / 5 계산에도 반영됨

14. 실드 규칙 최종 반영
- 기본 Defense 실드만 비교 대상
- 기존 실드 vs 새 기본 실드 중 큰 값 유지
- Unique 실드는 그 위에 별도 추가
- Attack Mode에서 얻는 Unique 실드도 바로 더함

예시 규칙
- 기존 실드 40, 새 기본 실드 25, Unique 실드 10
- 결과: max(40, 25) + 10 = 50

15. Attack Mode 처리 규칙 정리
- base result > 0:
  - 기본 공격 데미지 적용
- base result = 0:
  - 기본 데미지 없음
  - Unique 보너스 데미지는 별도 적용 가능
- base result < 0:
  - 적 회복 처리
  - Unique 3 보너스 데미지는 별도 적용 가능

16. 스테이지 클리어 골드 규칙 정리

19. 알려진 제한 / 검증 상태
- 현재 환경에 dotnet / csc / msbuild / mono 등 C# 빌드 도구가 없어 실제 컴파일 검증은 수행하지 못했다.
- 따라서 현재 검증은 코드 구조 확인, 참조 지점 확인, 정적 점검 중심이다.
